# ATBS 预测服务 - 模型训练与预测服务说明

## 决策依据摘要

本文档详细说明ATBS预测服务的技术实现，所有技术选型均基于以下决策依据：

| 决策项 | 决策结果 | 核心决策依据 |
|:------|:--------|:-------------|
| **预测模型** | ARIMA(2,1,1) | 精度(MAPE=5.8%)与成本的最佳平衡，训练速度比LSTM快10倍，可解释性强 |
| **参数搜索** | 网格搜索+5折交叉验证 | 生产环境必须考虑预测精度，交叉验证防止过拟合，并行计算加速 |
| **服务框架** | FastAPI+异步处理 | 高性能(接近Go)、原生异步支持、自动生成API文档、与Java生态兼容 |
| **数据预处理** | 三次样条插值+15%噪声 | 平滑趋势+模型鲁棒性，日度数据支持业务需求 |
| **评估指标** | MAPE(40%)+MAE(25%)+RMSE(20%)+AIC(15%) | 业务可理解性优先，综合评估模型性能与复杂度 |

> **详细决策依据**请参考《模型选型与优化决策依据.md》文档

```python
def find_best_params(

    """
```

### 2.3 网格搜索实现

```python
# training/grid_search.py - 网格搜索核心实现（第273-532行）

class TimeSeriesGridSearch:
    """
    时间序列网格搜索器（优化版本）
    实现网格搜索 + k-fold交叉验证 + 残差诊断 + 综合评估
    """

    def __init__(
        self,
        parameter_space: Optional[ParameterSpace] = None,
        cv_config: Optional[CrossValidationConfig] = None,
        scoring: str = 'mape',
        n_jobs: int = -1,
        verbose: int = 1,
        random_state: Optional[int] = None,
        use_composite_score: bool = True,
        residual_weight: float = 0.3
    ):
        """
        初始化网格搜索器
        
        Args:
            parameter_space: 参数搜索空间
            cv_config: 交叉验证配置
            scoring: 评分指标 ('mae', 'rmse', 'mape', 'aic', 'bic')
            n_jobs: 并行作业数，-1表示使用所有CPU
            verbose: 详细程度
            random_state: 随机种子
            use_composite_score: 是否使用综合评分
            residual_weight: 残差诊断在综合评分中的权重
        """
        self.parameter_space = parameter_space or ParameterSpace()
        self.cv_config = cv_config or CrossValidationConfig()
        self.scoring = scoring
        self.n_jobs = n_jobs if n_jobs > 0 else multiprocessing.cpu_count()
        self.verbose = verbose
        self.random_state = random_state
        self.use_composite_score = use_composite_score
        self.residual_weight = residual_weight

    def _cross_validate(
        self,
        data: pd.Series,
        order: Tuple[int, int, int]
    ) -> GridSearchResult:
        """
        对一组参数执行k-fold交叉验证
        
        Args:
            data: 时间序列数据
            order: ARIMA参数(p, d, q)
            
        Returns:
            网格搜索结果
        """
        # 创建时间序列分割器
        n_samples = len(data)
        test_size = self.cv_config.test_size
        n_splits = self.cv_config.n_splits
        
        fold_scores = []
        
        # 手动实现时间序列交叉验证
        for fold_idx in range(n_splits):
            # 计算训练集和测试集索引
            test_end = n_samples - fold_idx * (test_size + gap)
            test_start = test_end - test_size
            train_start = 0
            
            train_data = data.iloc[train_start:test_start]
            test_data = data.iloc[test_start:test_end]
            
            # 训练模型并评估
            scores = self._evaluate_single_fold(train_data, test_data, order)
            fold_scores.append(scores)
        
        # 计算平均值和标准差
        metrics = ['mae', 'rmse', 'mape', 'aic', 'bic']
        means = {m: np.mean([s[m] for s in valid_scores]) for m in metrics}
        stds = {m: np.std([s[m] for s in valid_scores]) for m in metrics}
        
        # 残差诊断
        residual_diagnostics = self._perform_residual_diagnostics(data, order)
        
        # 计算综合评分
        composite_score = self._calculate_composite_score(means, residual_diagnostics)
        
        return GridSearchResult(
            params=order,
            mean_mae=means['mae'],
            mean_rmse=means['rmse'],
            mean_mape=means['mape'],
            mean_aic=means['aic'],
            mean_bic=means['bic'],
            residual_diagnostics=residual_diagnostics,
            composite_score=composite_score
        )
```

### 2.4 平稳性检验

```python
# 平稳性检验（第279-295行）

def check_stationarity(self, data: pd.Series) -> Tuple[bool, float]:
    """
    使用ADF检验检查数据平稳性
    
    ADF检验（Augmented Dickey-Fuller Test）
    - 原假设H0: 序列存在单位根（非平稳）
    - 备择假设H1: 序列不存在单位根（平稳）
    - p值 < 0.05时拒绝原假设，认为序列平稳
    
    Args:
        data: 时间序列数据
        
    Returns:
        (是否平稳, p值)
    """
    result = adfuller(data.dropna())
    p_value = result[1]
    is_stationary = p_value < 0.05

    logger.info(f"ADF检验结果: p-value={p_value:.4f}, 平稳性={is_stationary}")

    return is_stationary, p_value
```

### 3.2 异步预测服务

```python
# app/services/async_prediction_service.py - 异步预测服务

class AsyncPredictionService:
    """
    异步预测服务
    支持异步训练和预测，避免阻塞主线程
    """
    
    async def predict(self, request: PredictionRequest) -> PredictionResult:
        """
        执行预测
        
        Args:
            request: 预测请求参数
            
        Returns:
            预测结果
        """
        try:
            # 检查模型是否存在
            if not self._check_model_exists():
                if request.auto_train:
                    # 自动训练模型
                    await self._auto_train()
                else:
                    return PredictionResult.error("模型不存在，请先训练模型")
            
            # 在线程池中执行预测（避免阻塞事件循环）
            loop = asyncio.get_event_loop()
            result = await loop.run_in_executor(
                self.executor,
                self._sync_predict,
                request
            )
            
            return PredictionResult.success(result)
            
        except Exception as e:
            logger.error(f"预测失败: {e}", exc_info=True)
            return PredictionResult.error(f"预测失败: {str(e)}")
    
    def _sync_predict(self, request: PredictionRequest) -> Dict:
        """
        同步预测方法（在线程池中执行）
        """
        predictor = ARIMAPredictor(settings.model_path)
        forecast_df = predictor.predict(
            steps=request.days,
            alpha=1 - request.confidence_level
        )
        
        return {
            'prediction_id': str(uuid.uuid4()),
            'prediction_time': DataProcessor.get_beijing_time().isoformat(),
            'model_info': predictor.get_model_info(),
            'forecasts': forecast_df.to_dict('records')
        }
```

## 4. 模型评估指标

### 4.1 评估指标说明（Table 4-1）

| 指标名称 | 英文全称 | 计算公式 | 说明 | 优秀标准 |
|:--------|:--------|:---------|:-----|:--------:|
| **MAE** | Mean Absolute Error | $\frac{1}{n}\sum_{i=1}^{n}\|y_i - \hat{y}_i\|$ | 平均绝对误差，反映预测值与真实值的平均偏差 | < 5% |
| **RMSE** | Root Mean Square Error | $\sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}$ | 均方根误差，对大误差更敏感 | < 10% |
| **MAPE** | Mean Absolute Percentage Error | $\frac{100\%}{n}\sum_{i=1}^{n}\|\frac{y_i - \hat{y}_i}{y_i}\|$ | 平均绝对百分比误差，无量纲 | < 15% |
| **AIC** | Akaike Information Criterion | $2k - 2\ln(L)$ | 赤池信息准则，衡量模型拟合优度 | 越小越好 |
| **BIC** | Bayesian Information Criterion | $k\ln(n) - 2\ln(L)$ | 贝叶斯信息准则，惩罚复杂模型 | 越小越好 |

### 5.2 数据预处理核心代码

```python
# utils/data_processor.py - 数据预处理核心实现（第413-564行）

def monthly_to_daily(
    self,
    interpolation_method: str = 'cubic',
    apply_effects: bool = True,
    apply_perturbation: bool = True,
    noise_level: float = 0.15,
    noise_type: str = 'adaptive',
    random_seed: Optional[int] = None
) -> pd.DataFrame:
    """
    将月度数据转换为日数据，确保每月日数据之和等于该月原始数据
    
    优化方案：
    1. 使用样条插值算法生成基础日度序列
    2. 加入增强的随机扰动以提高模型鲁棒性
    3. 整合星期几效应、月度内模式及行业特定规律
    4. 求和校验确保月度数据一致性
    
    Args:
        interpolation_method: 插值方法 ('cubic', 'pchip', 'linear')
        apply_effects: 是否应用多因素效应
        apply_perturbation: 是否应用随机扰动
        noise_level: 噪声水平（默认0.15）
        noise_type: 噪声类型（默认'adaptive'自适应）
        random_seed: 随机种子
        
    Returns:
        包含日数据的DataFrame
    """
    # 步骤1: 使用样条插值生成基础日度趋势
    daily_trend = self._interpolate_daily_trend(
        monthly_timestamps,
        monthly_values,
        daily_timestamps,
        interpolation_method
    )
    
    # 步骤2: 按月进行精细化处理
    for _, month_row in self.original_data.iterrows():
        month_period = pd.Period(month_row['Month'], freq='M')
        month_total = month_row['Passengers']
        
        # 获取该月的所有日数据
        month_mask = daily_df['YearMonth'] == month_period
        month_days = daily_df[month_mask].copy()
        
        # 步骤2a: 应用多因素效应
        if apply_effects:
            month_trend = self._apply_effects_to_month(
                month_days['Date'].values,
                month_trend
            )
        
        # 步骤2b: 应用随机扰动
        if apply_perturbation:
            month_trend = self.perturbation.apply_perturbation(
                month_trend,
                preserve_sum=False
            )
        
        # 步骤2c: 按比例分配确保总和等于月总量
        month_daily_values = self._distribute_monthly_total(
            month_total,
            month_trend,
            month_days['Trend'].values
        )
        
        daily_passengers.extend(month_daily_values)
    
    # 步骤3: 验证转换结果
    validation_result = self._validate_conversion()
    
    return self.daily_data
```

```python
# training/grid_search.py - 参数空间定义（第36-71行）

@dataclass
class ParameterSpace:
    """参数搜索空间定义（优化版本）"""
    
    # 基于航空客流数据的领域知识设定合理范围
    # p: 自回归阶数，通常2-4足够捕捉短期依赖
    # d: 差分阶数，通常0-2（ADF检验后确定）
    # q: 移动平均阶数，通常1-3足够
    
    p_values: List[int] = field(default_factory=lambda: [1, 2, 3, 4])
    d_values: List[int] = field(default_factory=lambda: [0, 1, 2])
    q_values: List[int] = field(default_factory=lambda: [0, 1, 2, 3])

    def get_all_combinations(self) -> List[Tuple[int, int, int]]:
        """获取所有参数组合"""
        valid_combinations = []
        for p, d, q in itertools.product(self.p_values, self.d_values, self.q_values):
            if not (p == 0 and q == 0):  # 排除无效模型
                valid_combinations.append((p, d, q))
        return valid_combinations
```

## 8. 关键技术决策说明

### 8.1 为什么选择ARIMA模型？

**决策对比**：

| 对比维度 | ARIMA | LSTM | 决策依据 |
|:--------|:------|:-----|:---------|
| 预测精度(MAPE) | 5.8% | 5.5% | 差距仅0.3%，在可接受范围内 |
| 训练时间 | 12.5秒 | 125.6秒 | ARIMA快10倍，利于快速迭代和部署 |
| 可解释性 | 高(p,d,q有明确含义) | 低(黑盒模型) | 业务方需要理解预测逻辑 |
| 部署复杂度 | 低(仅statsmodels依赖) | 高(TensorTorch/PyTorch) | 减少生产环境风险 |
| 维护成本 | 低(参数少，调试简单) | 高(超参数多，调优复杂) | 降低长期运维成本 |

**决策结论**：ARIMA在精度差距不大的情况下，综合成本最低，最适合生产环境。

### 8.2 为什么选择网格搜索+交叉验证？

**决策依据**：

1. **精度优先**：AIC准则仅考虑模型拟合优度，不直接反映预测精度
2. **防止过拟合**：交叉验证能有效评估模型泛化能力
3. **参数空间可控**：p≤5,d≤2,q≤5，共59种组合，网格搜索可行
4. **并行加速**：多进程并行可将搜索时间从45分钟缩短到12分钟

### 8.3 为什么选择FastAPI？

**决策对比**：

| 框架 | 性能 | 异步支持 | 类型安全 | 文档生成 | 决策 |
|:-----|:----:|:--------:|:--------:|:--------:|:----:|
| FastAPI | ★★★★★ | ✅ | ✅ | ✅ | ✅ |
| Flask | ★★★ | ❌ | ❌ | ❌ | ❌ |
| Django | ★★★ | ❌ | ❌ | ❌ | ❌ |
| Tornado | ★★★★ | ✅ | ❌ | ❌ | ❌ |

**决策依据**：FastAPI性能接近Go，原生支持异步，自动生成OpenAPI文档，与Java微服务生态完美兼容。

### 8.4 为什么需要异步训练？

**决策依据**：

1. ARIMA训练需要10-20秒，同步训练会阻塞HTTP请求
2. 异步训练立即返回任务ID，前端可轮询进度，提升用户体验
3. 支持并发训练多个模型，提高资源利用率
4. 实现复杂度适中，适合当前项目规模

## 9. 总结

ATBS预测服务是一个基于ARIMA模型的客流预测微服务，具有以下特点：

1. **算法先进**：采用网格搜索+交叉验证自动寻找最优ARIMA参数
2. **评估完善**：多维度评估指标（MAE、RMSE、MAPE、AIC、BIC）
3. **服务化**：FastAPI提供RESTful API，与Java微服务生态兼容
4. **异步处理**：支持异步训练和预测，避免阻塞
5. **自动重训**：支持定时自动重训和模型漂移检测
6. **数据预处理**：完善的月度到日度数据转换流程
