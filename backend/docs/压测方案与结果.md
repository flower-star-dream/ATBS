# ATBS 压力测试方案与结果

## 1. 压测目标

### 1.1 性能目标

| 指标 | 目标值 | 说明 |
|:-----|:------:|:-----|
| **吞吐量** | 5,000 QPS | 网关层总吞吐量 |

| **错误率** | < 0.1% | HTTP 5xx错误比例 |

| **内存使用率** | < 85% | 压测期间平均内存 |

## 3. 压测方案

### 3.1 测试场景设计

#### 场景1：基准测试（Baseline Test）

```
目的：获取系统基准性能数据
并发：100用户
持续时间：10分钟
Ramp-up：30秒

测试接口：
1. GET /api/app/v1/airplane/schedule/list - 航班列表查询
2. GET /api/app/v1/user/info - 用户信息查询
3. POST /api/app/v1/order/create - 订单创建

预期结果：
- 建立性能基线
- 识别性能瓶颈
```

#### 场景2：负载测试（Load Test）

```
目的：验证系统在正常负载下的表现
并发：500 → 1000 → 2000 → 5000用户（阶梯式增加）
持续时间：每个梯度5分钟
Ramp-up：60秒

测试接口：混合场景（按比例分配）
- 航班查询：60%
- 用户查询：20%
- 订单创建：20%

预期结果：
- 系统能稳定处理目标QPS
- 响应时间随负载线性增长
```

#### 场景3：压力测试（Stress Test）

```
目的：找到系统性能拐点
并发：持续增加直至系统崩溃
持续时间：直至错误率>5%或响应时间>10s
Ramp-up：持续增加

测试接口：核心接口压测
- 航班查询（高并发读取）
- 订单创建（高并发写入）

预期结果：
- 确定系统最大承载能力
- 识别系统崩溃临界点
```

#### 场景4：稳定性测试（Soak Test）

```
目的：验证系统长时间运行稳定性
并发：目标并发（5000用户）
持续时间：24小时
Ramp-up：5分钟

测试接口：混合场景

预期结果：
- 无内存泄漏
- 无连接池耗尽
- 性能无明显衰减
```

### 3.3 监控指标采集

```bash
#!/bin/bash
# 压测监控脚本

# 系统资源监控
while true; do
    echo "=== $(date) ===" >> monitor.log
    
    # CPU使用率
    echo "CPU Usage:" >> monitor.log
    top -bn1 | grep "Cpu(s)" >> monitor.log
    
    # 内存使用
    echo "Memory Usage:" >> monitor.log
    free -h >> monitor.log
    
    # 磁盘IO
    echo "Disk IO:" >> monitor.log
    iostat -x 1 1 >> monitor.log
    
    # 网络连接
    echo "Network Connections:" >> monitor.log
    netstat -an | grep :8080 | wc -l >> monitor.log
    
    # MySQL连接数
    echo "MySQL Connections:" >> monitor.log
    mysql -e "SHOW STATUS LIKE 'Threads_connected';" >> monitor.log
    
    # Redis连接数
    echo "Redis Connections:" >> monitor.log
    redis-cli INFO clients >> monitor.log
    
    sleep 5
done
```

## 6. 性能优化建议

### 6.1 已实施的优化

| 优化项 | 优化前 | 优化后 | 提升 |
|:------|:------:|:------:|:----:|
| 数据库连接池优化 | HikariCP默认配置 | 最大连接200，最小空闲20 | +15% |
| Redis缓存优化 | 无缓存 | 航班数据缓存5分钟 | +300% |
| JVM参数优化 | 默认配置 | G1GC，堆内存4G | +10% |
| SQL优化 | 全表扫描 | 添加复合索引 | +50% |

### 6.2 待优化项

| 优化项 | 预期效果 | 优先级 |

| 订单服务异步化 | 响应时间降低50% | 高 |
| 预测服务集群化 | 吞吐量提升3倍 | 中 |
| 数据库读写分离 | 查询性能提升2倍 | 中 |
| 引入消息队列 | 削峰填谷，降低系统压力 | 中 |
