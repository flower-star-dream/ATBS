## 决策依据说明

本文档说明实验数据集的选择依据和数据处理决策。

**核心决策问题**：为什么选择这个数据集？如何处理数据？

**关键决策点**：
1. 数据集选择：经典航空旅客数据集，具有代表性
2. 数据预处理：月度到日度转换，支持业务需求

## 1. 数据集概述

### 1.1 数据来源

本实验使用的航空客流数据来源于经典的航空旅客数据集（Airline Passengers Dataset），该数据集记录了1949年至1960年期间国际航空旅客的月度数量。

**数据集特点：**
- 时间跨度：12年（1949年1月 - 1960年12月）
- 数据频率：月度数据
- 数据点数量：144个观测值
- 数据单位：千人（Passengers in thousands）

### 1.2 数据集基本信息

| 属性 | 说明 |
|:-----|:-----|
| **数据集名称** | Airline Passengers Dataset |
| **时间范围** | 1949年1月 - 1960年12月 |
| **数据量** | 144条月度记录 |
| **数据格式** | CSV |
| **主要字段** | Month, Passengers |
| **数据单位** | 旅客数量（千人） |

## 5. 数据预处理

### 5.1 数据清洗

```python
# 数据清洗步骤

1. 缺失值检查
   - 原始数据完整性: 100% (无缺失值)
   - 日期连续性检查: 通过 (无间断)

2. 异常值检测
   - 使用IQR方法检测异常值
   - 结果: 未发现明显异常值
   - 所有数据点都在合理范围内

3. 数据类型转换
   - Month: String → DateTime
   - Passengers: String → Integer
```

### 5.2 月度到日度转换

由于ARIMA模型需要更高频率的数据进行训练，我们将月度数据转换为日度数据：

**转换方法：**
1. **样条插值**：使用三次样条插值生成基础日度趋势
2. **多因素效应**：应用星期效应、月度模式、季节效应
3. **随机扰动**：添加自适应噪声提高模型鲁棒性
4. **求和校验**：确保日度之和等于月度值

**转换结果：**
- 原始数据：144条月度记录
- 转换后：约4380条日度记录（12年×365天）

### 5.3 特征工程

```python
# 生成的特征

1. 时间特征
   - Year, Month, Day
   - DayOfWeek, DayOfYear
   - WeekOfYear, Quarter

2. 周期性特征（正弦余弦变换）
   - Month_Sin = sin(2π × Month / 12)
   - Month_Cos = cos(2π × Month / 12)
   - DayOfWeek_Sin = sin(2π × DayOfWeek / 7)
   - DayOfWeek_Cos = cos(2π × DayOfWeek / 7)

3. 滞后特征（用于模型训练）
   - Lag_1, Lag_2, ..., Lag_7
   - Rolling_Mean_7, Rolling_Mean_30
```

### 7.2 交叉验证策略决策依据

**决策问题：为什么选择时间序列交叉验证？**

| 验证策略 | 优点 | 缺点 | 适用场景 | 本决策 |
|:--------|:-----|:-----|:---------|:-------|
| **时间序列分割** | 保持时间顺序，符合实际 | 计算量大 | 时间序列数据 | ✅ 选择 |
| K折交叉验证 | 简单，数据利用率高 | 数据泄露，不符合时序特性 | 非时序数据 | ❌ 不适用 |
| 滑动窗口验证 | 模拟实际预测场景 | 实现复杂 | 在线学习 | ❌ 不适用 |
| 留出法 | 简单快速 | 数据利用率低 | 大数据集 | ❌ 不适用 |

**决策依据**：
1. 时间序列数据必须保持时间顺序，不能使用随机分割
2. 随机分割会导致未来信息泄露到训练集，造成过拟合
3. 时间序列分割能够模拟实际预测场景，提供更可靠的性能估计

采用时间序列交叉验证（Time Series Cross-Validation）：

```
Fold 1: [Train: 1949.01-1957.08] [Test: 1957.09-1958.08]
Fold 2: [Train: 1949.01-1958.08] [Test: 1958.09-1959.08]
Fold 3: [Train: 1949.01-1959.08] [Test: 1959.09-1960.08]
Fold 4: [Train: 1949.01-1959.12] [Test: 1960.01-1960.12]

特点：
- 保持时间顺序，不使用未来数据预测过去
- 训练集逐步扩大，测试集固定大小
- 更符合实际应用场景
```
